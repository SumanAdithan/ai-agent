# Environment Setup

Now that you know which models we will be using, you need to securely store your API keys and initialize them in your code.

<Callout type="error" title="Never Commit Your Keys!">
  Ensure that your `.gitignore` file contains `.env` so you do not accidentally push your private API keys to GitHub.
</Callout>

### The `.env` File
Create a `.env` file in the root of your project:

```properties title=".env"
OPENAI_API_KEY=your_openai_key_here
GOOGLE_API_KEY=your_gemini_key_here
GROQ_API_KEY=your_groq_key_here

# openai | gemini | groq
PROVIDER=gemini
```

### The `env.ts` File
Instead of calling `dotenv.config()` randomly across our application, we create a dedicated `src/env.ts` file. 

This file ensures that environment variables are loaded exactly **once** into the global `process.env` before any other logic runs, avoiding tricky race conditions.

```typescript title="src/env.ts"
import dotenv from "dotenv";

let loaded = false;

export function loadEnv(): void {
  if (loaded) return;
  
  dotenv.config();
  loaded = true;
}
```
