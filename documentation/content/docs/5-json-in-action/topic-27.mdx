# Defining the Zod Schema

Now that our project is initialized, the very first step in building a structured data agent is to define the **contract** that guarantees the shape of our data.

We do this by creating a Zod schema in our backend.

### 1. Installation

First, make sure you install `zod` in your `backend` directory:

```bash
pnpm add zod
```

### 2. The Schema File

Inside your `backend/src` directory, create a new file named `schema.ts`. This file will hold the exact structure we expect the LLM to output when answering a question.

```typescript title="backend/src/schema.ts"
import z from 'zod';

export const AskResultSchema = z.object({
    summary: z.string().min(1).max(1000),
    confidence: z.number().min(0).max(1)
});

export type AskResult = z.infer<typeof AskResultSchema>;
```

### 2. Why this structure?

Let's break down exactly what this schema is enforcing:

*   **`summary`**: A string that contains the actual answer generated by the LLM. By setting `.min(1).max(1000)`, we are adding runtime validation to ensure the LLM doesn't return an empty string or generate an excessively long, runaway answer.
*   **`confidence`**: A numeric score representing how confident the LLM is in its generated answer. By setting `.min(0).max(1)`, we enforce that this must be a float between 0.0 and 1.0 (e.g., `0.95`).

### 3. TypeScript Inference

The magic of Zod happens in the final line:

```typescript
export type AskResult = z.infer<typeof AskResultSchema>;
```

This single line tells TypeScript to read our runtime Zod schema and automatically generate a static TypeScript interface for it. Whenever we handle the output of the LLM later in our Express routes, we will have perfect autocomplete and type safety for `summary` and `confidence`!
