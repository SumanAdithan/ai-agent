# Integrate Search Agent with Next.js

Our LangChain search agent is fully operational and exposed via an Express API. The final step of this project is to build a modern React UI using Next.js to interact with our agent!

### The Implementation

First, ensure your Next.js application knows where to find the Express API by setting up your environment variables. In Next.js, environment variables that need to be accessible in the browser **must** be prefixed with `NEXT_PUBLIC_`.

```typescript title="client/src/lib/config.ts"
export const API_URL = process.env.NEXT_PUBLIC_API_URL;
```

Next, open your main UI component at `client/src/app/page.tsx` and replace its contents with the following:

```tsx title="client/src/app/page.tsx"
"use client";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";
import { API_URL } from "@/lib/config";
import Link from "next/link";
import { FormEvent, SubmitEvent, useEffect, useRef, useState } from "react";

type SearchResponse = {
  answer: string;
  sources: string[];
};

type CurrentChatTurn =
  | {
      role: "user";
      content: string;
    }
  | {
      role: "assistant";
      content: string;
      sources: string[];
      time: number;
      error?: string;
    };

export default function Home() {
  const [query, setQuery] = useState("");
  const [loading, setLoading] = useState(false);
  const [chat, setChat] = useState<CurrentChatTurn[]>([]);

  const scrollRef = useRef<HTMLDivElement | null>(null);

  // Auto-scroll to the bottom when new messages arrive!
  useEffect(() => {
    scrollRef.current?.scrollTo({
      top: scrollRef.current.scrollHeight,
      behavior: "smooth",
    });
  }, [chat]);

  async function runSearch(prompt: string) {
    setLoading(true);
    setChat((old) => [...old, { role: "user", content: prompt }]);
    const oldTime = performance.now();

    try {
      // 1. Call our Express LCEL Backend!
      const res = await fetch(`${API_URL}/search`, {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
        },
        body: JSON.stringify({ q: prompt }),
      });

      const json = await res.json();
      const timeDiff = Math.round(performance.now() - oldTime);

      if (!res.ok) {
        throw new Error(json.error || "Request failed");
      }

      // 2. Safely cast the response using our defined types
      const data = json as SearchResponse;
      
      // 3. Append the AI's response to the chat log
      setChat((old) => [
        ...old,
        {
          role: "assistant",
          content: data.answer,
          sources: data.sources,
          time: timeDiff,
        },
      ]);
    } catch (error) {
      const timeDiff = Math.round(performance.now() - oldTime);
      setChat((old) => [
        ...old,
        {
          role: "assistant",
          content: "I tried to answer, but something went wrong. Please try again.",
          sources: [],
          time: timeDiff,
          error: "Request failed",
        },
      ]);
    } finally {
      setLoading(false);
    }
  }

  async function handleChatSubmit(e: SubmitEvent<HTMLFormElement>) {
    e.preventDefault();
    const prompt = query.trim();

    if (!prompt || loading) return;
    setQuery("");

    await runSearch(prompt);
  }

  return (
    <div className="flex h-dvh flex-col bg-[#f9fafe] text-gray-900">
      <header className="border-b bg-white px-4 py-3 text-sm flex items-center justify-between">
        <div className="flex flex-col">
          <span className="font-medium text-gray-800">
            Search V1 (LCEL Web Agent)
          </span>
          <span className="text-[12px] text-gray-500">
            Answer with sources. Some queries will browse the web and some don't.
          </span>
        </div>
      </header>
      
      <main className="flex-1 overflow-y-auto px-4 py-6 space-y-6">
        {/* Render Chat Messages Here */}
        {chat.map((turn, idx) => {
          if (turn.role === "user") {
            // Render User Bubble
            return (
              <div key={idx} className="mx-auto max-w-2xl flex justify-end text-right">
                <div className="inline-block rounded-2xl bg-gray-900 px-4 py-3 text-sm text-white shadow-md max-w-full">
                  <div className="whitespace-pre-wrap wrap-break-words">{turn.content}</div>
                </div>
              </div>
            );
          }
          
          // Render AI Bubble
          return (
            <div key={idx} className="mx-auto max-w-2xl flex items-start gap-3 text-left">
              <div className="flex h-8 w-8 flex-none items-center justify-center rounded-md bg-gray-800 text-[11px] text-white font-semibold">
                AI
              </div>
              <div className="flex-1 space-y-3">
                <div className="inline-block rounded-2xl bg-white px-4 py-3 text-sm text-gray-900 shadow-sm ring-1 ring-gray-200 whitespace-pre-wrap wrap-break-word">
                  {turn.content}
                </div>
                
                {/* Dynamically render sources if they exist! */}
                {turn.sources && turn.sources.length > 0 && (
                  <div className="rounded-lg bg-white px-3 py-2 text-[12px] shadow-sm ring-gray-200">
                    <div className="text-[11px] font-medium text-gray-600 mb-1">Sources</div>
                    <ul className="space-y-1">
                      {turn.sources.map((source, idx) => (
                        <li key={idx} className="truncate">
                          <Link href={source} target="_blank" className="text-blue-500 underline underline-offset-4 break-all">
                            {source}
                          </Link>
                        </li>
                      ))}
                    </ul>
                  </div>
                )}
              </div>
            </div>
          );
        })}
      </main>

      <footer className="border-t bg-white px-4 py-4">
        <form onSubmit={handleChatSubmit} className="mx-auto flex w-full max-w-2xl items-end gap-2">
          <Input
            className="w-full resize-none"
            placeholder="Ask your query..."
            value={query}
            onChange={(event) => setQuery(event.target.value)}
            disabled={loading}
          />
          <Button disabled={loading || query.trim().length < 5} type="submit">
            {loading ? "..." : "Send"}
          </Button>
        </form>
      </footer>
    </div>
  );
}
```

### Breaking it down

<Callout title="The Beauty of Shared Contracts" type="info">
  Because we aggressively utilized Zod schemas on the backend to enforce the shape of `SearchAnswerSchema`, our frontend developer experience is incredible! We can simply define a TypeScript type (`SearchResponse`) that matches that schema, cast our API response directly to it, and render the UI without writing any complex error-checking logic!
</Callout>

1. **The Chat State:** We maintain a `chat` array of type `CurrentChatTurn[]`. This allows us to render a full conversational history.
2. **Conditional Source Rendering:** Thanks to our LCEL `branch` logic, some queries ("direct mode") return an empty sources array, while others ("web mode") return actual URLs. Our UI gracefully handles this with `{turn.sources && turn.sources.length > 0 && (...) }`. The UI doesn't care *how* the agent got the answer; it just renders what it was given!
3. **Performance Tracking:** We wrapped the `fetch` call in `performance.now()` markers. If you ask a simple question (like "What is 2+2?"), the agent takes the `directPath` and answers in ~800ms. If you ask a complex web query (like "Top 10 colleges 2025"), the agent takes the `webPath`, scrapes the internet, and answers in ~4000ms. The UI dynamically shows this time difference to the user!

Congratulations! You have successfully built a full-stack, LCEL-powered Web Search Agent!
